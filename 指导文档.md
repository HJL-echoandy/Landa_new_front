基于 Expo 的调试非常直观，主要有以下几种方式，从简单到高级：

### 1. 📱 手机端：摇一摇（Shake）
这是最常用的方式。
*   **操作**: 确保手机和电脑在同一 WiFi 下，运行 `npx expo start`，手机用 Expo Go 扫码进入 App 后，**用力摇晃手机**。
*   **弹出菜单**:
    *   **Reload**: 刷新页面。
    *   **Toggle Element Inspector**: 开启 UI 检查模式（类似浏览器的审查元素）。
    *   **Open JS Debugger**: 开启远程 JS 调试。

### 2. 💻 终端控制台 (Terminal)
运行 `npx expo start` 的那个黑色窗口本身就是控制台。
*   **日志**: 代码里的 `console.log('Hello')` 会直接打印在这里。
*   **快捷键**:
    *   `r`: 重新加载 (Reload)。
    *   `m`: 打开手机上的开发者菜单 (Menu)。
    *   `j`: 打开 JS 调试器。

### 3. 🌐 浏览器调试 (JS Debugger)
当你需要打断点 (Breakpoint) 或者看对象详情时。
*   **操作**:
    1.  在终端按 `j`，或者在手机菜单选 `Open JS Debugger`。
    2.  这会打开一个 Chrome 窗口 (Hermes Debugger)。
    3.  按 `F12` 打开开发者工具。
    4.  去 **Sources** 面板，按 `Ctrl+P` 搜索你的文件名（比如 `App.js`）。
    5.  点击行号打断点。
    6.  手机触发操作，Chrome 会自动暂停在断点处，你可以看变量值。

### 4. 🕵️‍♂️ UI 检查器 (Element Inspector)
当你想看布局、间距、样式对不对时。
*   **操作**: 在手机菜单里开启 `Toggle Element Inspector`。
*   **效果**: 点击手机屏幕上的组件，会显示它的宽、高、Margin、Padding、颜色等信息。

### 5. 🛠️ React Native Tools (VS Code 插件) - 推荐
如果你不想离开 VS Code。
*   安装 **"React Native Tools"** 插件。
*   你可以直接在 VS Code 代码行号旁边点红点打断点。
*   配置 `launch.json` 后，可以直接在 VS Code 里启动调试，变量值会直接显示在编辑器里。

---

**总结推荐流程：**
1.  **日常**: 看终端里的 `console.log`。
2.  **UI 歪了**: 手机摇一摇开 `Element Inspector`。
3.  **逻辑错了**: 终端按 `j` 调出 Chrome 打断点。
